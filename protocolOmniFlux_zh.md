## 帧格式

所有 SDK 帧与 DFT SD101 / AirWave 系列采用相同的二进制布局：

| 字节偏移 | 字段     | 说明                                   |
| -------- | -------- | -------------------------------------- |
| 0–1      | 帧头     | 固定 `0xDF 0xDF`                       |
| 2        | Func     | 功能码（见下文各节）                   |
| 3        | Cmd      | 该功能下的命令序号                     |
| 4        | Len      | 载荷长度（字节）                       |
| 5…(4+Len) | Data    | 载荷（Len = 0 时可为空）               |
| 5+Len    | 校验和   | 帧内此前所有字节之和的低 8 位          |

- 结构化载荷中的多字节数值字段默认采用**小端序**，除非另有说明。
- `float` 为 IEEE-754 单精度（4 字节，小端）。
- 设备侧单字节接收超时：**100 ms**（超时后须重新以 `0xDF 0xDF` 识别新帧）。

### 版本 1.0.2（2026 年 6 月 9 日）

**新增能力：**

1. **设备设置 - 系统与烘焙参数（Cmd 2–16）**
   - 自动关机、亮度、时间格式、蜂鸣器、USB 灯、在线烘焙相关开关及色值/温度单位等。
   - 读写语义与 Cmd 0（语言）一致：`Len=0` 读取当前值，`Len=1` 设置并返回当前值（非法值不改变）。

2. **设备动作 - 在线烘焙 SDK 精简（Func 3，Cmd 6–14）**
   - 以结构体为单位查询：实时包、环境量、全部关键点；不再提供单项 float / TOF / 入出豆事件等分散 Cmd。
   - 推送订阅改为 Cmd **14**（原 Cmd 29）。

### 版本 1.0.1（2026 年 6 月 1 日）

**新增能力：**

> **说明**：OmniFlux（CRM101）在蓝牙拓扑中作为**主机**，对接 AirWave 等外设。第三方客户端也可在 AA01 特征上使用相同帧格式读取烘焙数据或遥控烘焙界面。

1. **设备设置 - 蓝牙名称（Cmd 1）**
   - 返回 CRM101 广播的 BLE 名称（通常以 `OmniFlux` 为前缀）。
   - 应答后，设备可向已连接的 **AirWave** 主动下发进入自动模式的指令（见下文 AirWave 联动）。

2. **设备动作 - AirWave 烘焙阶段（Cmd 4）**
   - 兼容 AirWave 的查询：返回联动阶段字节 `is_start`（映射到 AirWave 烘焙阶段枚举）。

3. **设备动作 - 在线烘焙 SDK（Cmd 6 起）**（v1.0.2 起见 Func 3 详表）
   - 实时烘焙遥测、环境量、关键点、远程 UI 控制及周期推送。

**AirWave 联动流程：**

1. **连接**：AirWave 经蓝牙连接 CRM101（SDK 通道 AA01）。
2. **名称查询**：AirWave 发送设备设置 Cmd 1（载荷长度为 0）。
3. **名称应答**：CRM101 回复 BLE 名称（`OmniFlux…`）。若名称以 `OmniFlux` 开头，AirWave 识别对端为 OmniFlux。
4. **自动模式**：名称应答约 **2 s** 后，CRM101 向 AirWave 发送 `DF DF 03 05 01 01 C8`（设备动作 Cmd 5，设为自动模式）。
5. **阶段轮询**：自动模式下，AirWave 以 `DF DF 03 04 00 C5` 轮询烘焙阶段；CRM101 以 `is_start` 应答（Func 3，Cmd 4）。

## 3. 设备动作 / 烘焙检测（Func 3）

Func `3` 承载在线烘焙遥测、UI 遥控及 AirWave 联动。每个数据类 Cmd 对应一个完整结构体，不再提供单项 float 分散查询。

**方向说明：** `Q` = 查询（零长度载荷，设备应答），`W` = 写入/控制，`P` = 设备主动推送（订阅后）。

**Cmd 分配总览：**

| Cmd | 说明 | 方向 |
| --- | ---- | ---- |
| 4 | AirWave 联动阶段查询 | `Q` |
| 5 | AirWave 自动模式（**设备主动下发**，客户端无需发送） | — |
| 6 | 完整实时数据包 | `Q` / `P` |
| 7 | 环境传感器数据 | `Q` |
| 8 | 全部关键点（转黄 / 一爆 / 二爆） | `Q` |
| 9 | 烘焙运行状态 | `Q` |
| 10 | 开始 / 停止烘焙曲线 | `W` |
| 11 | 标记转黄点 | `W` |
| 12 | 标记一爆 | `W` |
| 13 | 标记二爆 | `W` |
| 14 | 实时推送订阅控制 | `W` |

### 3.1 AirWave 联动（Cmd 4 / 5）

|             | Func | 功能说明   | Cmd | 命令说明           | 数据长度 | 数据说明           | 备注 |
| ----------- | ---- | ---------- | --- | ------------------ | -------- | ------------------ | ---- |
| 控制命令    | 3    | 设备动作   | 4   | 读取联动/烘焙阶段  | 0        | /                  | AirWave「询问烘焙阶段」 |

**`is_start` 与 AirWave 烘焙阶段映射（CRM101 实现）：**

| `is_start` | CRM101 含义    | AirWave 阶段      |
| ---------- | -------------- | ----------------- |
| 0          | 空闲 / 等待    | Wait（0）         |

> Cool（5）当前不由 `is_start` 驱动；降温阶段请结合 UI 状态判断。  
> Cmd 9（烘焙运行状态）返回的 `is_start` 语义与 Cmd 4 相同。

### *示例（AirWave 阶段轮询）：*

- AirWave 命令：`DF DF 03 04 00 C5`
- CRM101 应答：`DF DF 03 04 01 02 C8` → `0x02` = 转黄（Turn Yellow）

| 应答        | 3    | 设备动作   | 10  | 返回当前状态     | 1        | 当前 `is_start`                    |      |
| 控制命令    | 3    | 设备动作   | 11  | 标记转黄点       | 0        | /                                  | `W`  |
| 应答        | 3    | 设备动作   | 11  | 返回执行结果     | 1        | `1` = 已接受                       |      |
| 控制命令    | 3    | 设备动作   | 12  | 标记一爆         | 0        | /                                  | `W`  |
| 应答        | 3    | 设备动作   | 12  | 返回执行结果     | 1        | `1` = 已接受                       |      |
| 控制命令    | 3    | 设备动作   | 13  | 标记二爆         | 0        | /                                  | `W`  |
| 应答        | 3    | 设备动作   | 13  | 返回执行结果     | 1        | `1` = 已接受                       |      |

### *示例：*

- 开始 / 切换烘焙曲线
  - 命令：`DF DF 03 0A 01 01 CD`（Cmd 10 = `0x0A`，数据 `0x01`）
  - 应答：`DF DF 03 0A 01 … 当前 is_start … CS`
- 标记转黄点（无载荷）
  - 命令：`DF DF 03 0B 00 CC`
  - 应答：`DF DF 03 0B 01 01 CE`

### 3.4 周期推送订阅（Cmd 14）

|             | Func | 功能说明   | Cmd | 命令说明     | 数据长度 | 数据说明                                    | 备注 |
| ----------- | ---- | ---------- | --- | ------------ | -------- | ------------------------------------------- | ---- |
| 控制命令    | 3    | 设备动作   | 14  | 推送订阅控制 | 3        | `sdk_det_push_ctrl_t`                       | `W`  |
| 应答        | 3    | 设备动作   | 14  | 返回使能状态 | 1        | `0`=已停止 `1`=推送中                       |      |
| 推送        | 3    | 设备动作   | 6   | 实时烘焙推送 | 38       | 同 Cmd 6 应答（`sdk_det_roast_realtime_t`） | `P`  |

**`sdk_det_push_ctrl_t`（3 字节，packed）：**

```c
typedef struct {
    uint8_t  enable;       // 0=停止  1=开启
    uint16_t interval_ms;  // 推送间隔 (ms)，最小 100
} sdk_det_push_ctrl_t;
```

### *示例：*

- 开启推送，间隔 500 ms
  - 命令：`DF DF 03 0E 03 01 F4 01 C8`（Cmd 14 = `0x0E`，enable=`1`，interval=`500` 小端）
  - 应答：`DF DF 03 0E 01 01 D1`
  - 之后设备周期性发送 Cmd **6** 帧，载荷 38 字节实时包。

- 关闭推送
  - 命令：`DF DF 03 0E 03 00 00 00 D2`
  - 应答：`DF DF 03 0E 01 00 D0`
